Reference

  • Integreat CMS: The main package of the integreat-cms with the following sub-packages:

    • API: This is the app which contains all API routes and Classes which maps the cms models to API JSON responses. This is not the API documentation itself, but the Django developer documentation. A link to the API documentation will follow soon.

    • CMS: This is the content management system for backend users which contains all database models, views, forms and templates.

    • Core: This is the project’s main app which contains all configuration files.

    • Gvz API: This is the app to communicate with our Gemeindeverzeichnis API to automatically import coordinates and region aliases

    • Nominatim API: This is the app to communicate with our Nominatim API to automatically import region bounding boxes

    • Sitemap: This is the app to dynamically generate a sitemap.xml for the webapp

    • XLIFF: The XLIFF serializer module

  • Tests: The tests for integreat-cms
